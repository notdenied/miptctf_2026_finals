1) SSRF через mPDF:

https://github.com/brun0ne/mpdf-pocs
https://medium.com/@brun0ne/breaking-mpdf-with-regex-and-logic-bf915300483f

Она:
- Не считается багом (SSRF через `@import url(...)`) и потому не пофикшена в новых версиях.
- Полная (можно использовать gopher://).
- Тяжело фиксится (скорее всего, её сначала просплоитят через `<img src=...>`, потом команды накинут экранирование html, но это не поможет).

С ней можно сходить во внутренние ручки (особо юзлесс) или положить таски в редиску (вот это уже полезно). Таска достанет условия содержания, в которых есть флаг. Но, вообще говоря, простор для эксплуатации был большой -- попробовать записать PHP шелл через redis, php-фильтры и т. д.

2) Кривой rewrite для защиты internal ручек на Nginx -> можем обратиться к ним. Например, если в урле будет %0a, то правило реврайта не применится, потому что оно применяется уже к декодированному углу (и регулярка метчит от начала до конца строки).

Можем получить PENDING таски, которые лежат там минимум секунд 8 (соврал, 10)...

В них есть записки исследователя с флагом.
